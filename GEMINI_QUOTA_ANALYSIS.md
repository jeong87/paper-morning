# Gemini API (Free Tier) 할당량 분석 및 `paper-morning` 호환성 평가

> **작성일**: 2026-03-05  
> **기준 환경**: `paper-morning v0.2.7` 기본 설정 (하루 1회 자동 실행)

---

## 1. `paper-morning`의 1회 실행 시 API 호출량 분석

현재 코드(`paper_digest_app.py`의 `annotate_papers_with_llm` 및 `.env` 기본값) 기준, API 호출량은 다음과 같이 계산됩니다.

### 설정 기본값 (`.env.example`)
*   `LLM_MAX_CANDIDATES=20` (LLM 평가에 넘길 최대 논문 수)
*   `LLM_BATCH_SIZE=5` (한 번의 API 호출에 묶어보낼 논문 수, v0.2.6에서 수정됨)

### 1회 실행(Daily Run) 소모량 계산
1.  **API 호출 횟수(Requests)**: 
    *   `MAX_CANDIDATES` (20) / `BATCH_SIZE` (5) = **최대 4회 호출**
2.  **토큰 사용량(Tokens) 추정**:
    *   논문 1편당 초록 길이는 최대 1500자로 제한됨 (`clean_text(paper.abstract)[:1500]`)
    *   1편당 대략 500~700 토큰 발생
    *   1 API 호출 (5편 묶음) = 프롬프트 지시문 + (논문 5편) = **약 3,000 ~ 4,000 입력 토큰**
    *   출력 토큰 = 평가 점수 1개 + 한국어 3줄 요약 x 5편 = **약 500 출력 토큰**
    *   **하루 전체(4회 호출) 토큰 방출량** = **약 12,000 ~ 16,000 입력 토큰** / **2,000 출력 토큰**

## 2. Google AI Studio 무료 티어(Free Tier) 할당량 비교

2026년 기준 Google AI Studio 무료 티어 제한은 대체로 다음과 같습니다 (모델 세대별로 세부 차이는 있으나 큰 틀은 유사함).

### A. 공통 무료 티어 제한 (대략적 기준)
*   **RPM (Requests Per Minute)**: 분당 15회
*   **RPD (Requests Per Day)**: 일일 1,500회
*   **TPM (Tokens Per Minute)**: 분당 1,000,000 토큰 (1M)

### B. 모델별 호환성 및 권장도

| :--- | :--- | :--- | :--- |

## 3. 극단적 엣지 케이스 (Maximum 상황)

### "만약 사용자가 설정을 극한으로 올린다면?"
*   사용자 임의 설정: `LLM_MAX_CANDIDATES=50`, `LLM_BATCH_SIZE=2`
*   발생하는 일: 1회 실행 시 **API 25번 호출**
*   **결과**: API 25번을 for-loop로 순차 호출하므로 대략 2~3분 소요. 
*   **영향**: 분당 15회 제한(15 RPM)에 걸려 중간에 `429 Too Many Requests` 에러가 날 확률이 높습니다. 

### "앱 자체 방어 로직 반영 결과"
현재 `paper-morning v0.2.7`은 고맙게도 기본 `LLM_BATCH_SIZE`를 8에서 5로 낮추었습니다. (v0.2.6 변경사항)
*   일반적으로 논문 초록이 길면 토큰 초과(4000 토큰 제한 등)가 날 수 있는데, **배치를 5로 줄였기 때문에 한 번 호출의 토큰 사이즈가 안전권(3000 언저리)에 들어왔습니다.**
*   `LLM_MAX_CANDIDATES`가 20이므로 하루 딱 **4번만** 부릅니다. 
*   구글의 허들(분당 15번)에 절대 부딪히지 않습니다.

## 4. 결론

**하루 1번 실행하는 일반적인 용도라면, 질문하신 모든 Gemini 모델(2.5/3.0/3.1 시리즈)의 무료 티어(Free Tier)로 차고 넘칩니다.** 
개발자 카드 등록(과금)을 하지 않고 무료 API 키만 발급받아도 안정적으로, 에러 없이 수 개월간 매일 사용할 수 있는 아주 경제적이고 훌륭한 구조입니다.
