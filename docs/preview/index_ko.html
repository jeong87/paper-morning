내 연구 맥락에 실제로 맞는 논문만 빠르게 찾습니다.

연구 주제를 설명하고 Gemini API key를 넣으면, Paper Morning이 맥락 기반 검색 쿼리를 만들고 실제 후보 논문을 가져온 뒤 관련도를 정리해 새 탭 결과 페이지를 열어줍니다. 이 페이지에서는 이메일이 발송되지 않습니다.

컨텍스트 우선 복잡한 검색식보다, 지금 하고 있는 연구 설명부터 시작합니다.
실제 논문 수집 프리뷰가 arXiv와 PubMed에서 실시간 후보를 가져온 뒤 순위를 매깁니다.
검색 모드 목적에 따라 최신 동향, 정확도 우선, 탐색 확장 모드를 바꿔가며 볼 수 있습니다.
설치 없이 체험 브라우저에서 바로 돌아가므로 GitHub 방문자가 제품 가치를 먼저 확인하기 좋습니다.

라이브 검색 실행

GitHub에서 들어와 제품이 무엇을 하는지 빠르게 확인하고 싶을 때 이 페이지를 사용하세요. Gemini API key는 이번 실행에만 사용됩니다. 마지막으로 생성된 HTML 결과만 브라우저 로컬 저장소에 보관하므로 다시 검색하지 않고도 마지막 결과를 다시 열 수 있습니다.

선택 입력입니다. 비워두면 컨텍스트를 바탕으로 제목을 자동 생성합니다.
필수 입력입니다. 컨텍스트가 구체적일수록 검색식 생성과 랭킹 품질이 좋아집니다.
쉼표로 구분해 입력하세요. 컨텍스트 기반 검색식 생성에 함께 반영됩니다.
이번 주 논문이 아니어도, 지금 프로젝트에 가장 잘 맞는 논문을 우선 찾습니다.
정확도 우선 모드의 기본값은 1년입니다. 기간이 길수록 더 넓게 검색하며, 더 강한 Gemini 모델이 유리할 수 있습니다.

준비되었습니다. 연구 컨텍스트를 입력하고 검색 모드를 고른 뒤 Gemini key를 넣어 검색을 실행하세요.

실행 버튼을 누르면

  1. 연구 컨텍스트에서 검색식을 만듭니다 먼저 Gemini가 프로젝트 설명을 읽고 토픽 키워드와 arXiv, PubMed용 검색식을 생성합니다.
  2. 선택한 목적에 맞춰 후보 논문을 수집합니다 최신 동향은 최근성을 우선하고, 정확도 우선과 탐색 확장은 더 넓은 기간에서 컨텍스트 적합도를 높게 봅니다.
  3. Gemini가 후보를 다시 평가합니다 이어서 Gemini가 후보들을 같은 기준으로 비교해 관련성 점수를 매기고, 왜 중요한지 짧게 설명합니다.
  4. 결과 페이지를 새 탭으로 엽니다 정렬된 결과를 바로 확인할 수 있지만, 이 페이지에서는 이메일이 발송되지 않습니다.

빠르지만 유연합니다. 5개 결과 선정, arXiv/PubMed 실검색, 검색 목적과 기간 조절, 이메일 발송 없음.

브라우저 한계 때문에 이 페이지는 가장 빠른 무설치 체험 경로에 집중합니다. 로컬 설치를 하면 더 많은 운영 옵션을 쓸 수 있지만, 이 라이브 검색만으로도 제품 가치가 충분히 드러나도록 구성했습니다.
